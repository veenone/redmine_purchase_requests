# User Guide: Redmine Purchase Requests Plugin

## Introduction

This guide provides step-by-step instructions for using the Purchase Requests Plugin in Redmine.

## Creating a Purchase Request

### Steps to Create a Request

1. Navigate to Purchase Requests
2. Click "New Purchase Request"
3. Fill in required fields:
   - Title
   - Description
   - Budget Category (CAPEX/OPEX)
   - Estimated Cost
   - Currency
   - Priority
   - Vendor (optional)

### Request Workflow

```
Draft → Pending Approval → Approved → In Progress → Completed
```

## Managing Purchase Requests

### Dashboard Features
- View all your purchase requests
- Filter by status, budget category, date
- Sort and search functionality

### Request Statuses
- **Draft**: Initial request creation
- **Pending Approval**: Awaiting management review
- **Approved**: Request authorized
- **In Progress**: Purchase being processed
- **Completed**: Request fulfilled
- **Rejected**: Request not approved

## Budget Tracking

### CAPEX vs OPEX
- **CAPEX**: Capital expenditures for long-term assets
- **OPEX**: Operational expenditures for ongoing costs

### Budget Allocation
- View quarterly budget allocations
- Track spending against budget
- Receive alerts for budget thresholds

## Vendor Management

- Add and manage vendor information
- Compare vendor quotes
- Track vendor performance

## Reporting

### Available Reports
- Quarterly budget utilization
- Vendor spending analysis
- Request status distribution
- Cost breakdown by category

## Notifications

- Email notifications for:
  - Request status changes
  - Approval requirements
  - Budget threshold alerts

## Best Practices

1. Provide detailed descriptions
2. Attach relevant documents
3. Select appropriate budget category
4. Update request status promptly
5. Communicate with approvers

## Permissions

- View own requests
- Create new requests
- Edit draft requests
- Submit for approval

## Troubleshooting

- **Cannot create request**: Check permissions
- **Missing fields**: Ensure all required information is provided
- **Budget concerns**: Consult with finance team
